# Commit Message 规范

本项目使用 [Conventional Commits](https://www.conventionalcommits.org/) 规范。

## 格式

```
<type>(<scope>): <subject>

<body>

<footer>
```

## Type 类型

必须是以下之一：

- **feat**: 新功能 (feature)
- **fix**: 修复 bug
- **docs**: 文档变更
- **style**: 代码格式（不影响代码运行的变动）
- **refactor**: 重构（既不是新增功能，也不是修复 bug）
- **perf**: 性能优化
- **test**: 增加测试
- **build**: 构建过程或辅助工具的变动
- **ci**: CI 配置文件和脚本的变动
- **chore**: 其他不修改 src 或测试文件的变动
- **revert**: 回滚 commit

## Scope 范围（可选）

用于说明 commit 影响的范围，比如：

- **components**: 组件相关
- **utils**: 工具函数
- **theme**: 样式主题
- **docs**: 文档
- **build**: 构建配置
- **test**: 测试相关

## Subject 主题

- 使用祈使句，现在时态："change" 而不是 "changed" 或 "changes"
- 不要大写首字母
- 结尾不要加句号

## 示例

### 新功能
```
feat(components): add Button component

- Add basic button component
- Support different types and sizes
- Add loading state
```

### 修复 Bug
```
fix(utils): fix BEM modifier function

Fixed the m() function that was hardcoding 'modifier' string
```

### 文档更新
```
docs: update README with installation guide
```

### 代码格式
```
style(components): format button component code
```

### 重构
```
refactor(utils): simplify createNamespace function
```

### 性能优化
```
perf(components): optimize virtual list rendering
```

### 测试
```
test(components): add Button component tests

- Add 12 test cases for Button
- Test props, events, and slots
```

### 构建配置
```
build: add vite build configuration

- Configure library mode
- Add type declaration generation
```

### CI 配置
```
ci: add GitHub Actions workflow
```

### 其他变更
```
chore: update dependencies
```

### 回滚
```
revert: revert "feat: add new feature"

This reverts commit abc123.
```

## 规则

1. **type** 必须小写
2. **type** 不能为空
3. **subject** 不能为空
4. **subject** 结尾不能有句号
5. **header** 最大长度 100 字符

## 验证

提交时会自动验证 commit message 格式，不符合规范的提交会被拒绝。

可以使用以下命令手动验证：

```bash
echo "feat: add new feature" | npx commitlint
```
